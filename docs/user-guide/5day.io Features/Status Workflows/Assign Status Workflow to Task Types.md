## Overview

Assigning a **Status Workflow** to a Task Type determines how tasks of that type will be tracked across projects.
## **How to Assign a Status Workflow to a Task Type**
1. Log in to [**5day.io**](http://5day.io/).
1. Go to the workspace where the Task Type exists.
1. Click the **three-dot (⋮) icon** next to the workspace name.
1. Click **Workspace Settings**.
1. Click **Customization**.
1. Click **Task Type**.
1. Select the Task Type for which you want to assign or change the Status Workflow.
1. Under **Status Workflow**, select the required workflow from the dropdown.
1. Click **Save**.

**Note:** If the Task Type is already being used in any project and tasks exist under it, the system shows a **Remap Status** screen.

In the Remap Status screen, select the appropriate new status for each old status and click **Save**.

If no tasks exist under the Task Type, the workflow will change without showing the Remap Status screen.
