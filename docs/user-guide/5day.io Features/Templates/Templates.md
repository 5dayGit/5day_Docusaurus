## Overview

**Templates** in [5day.io](http://5day.io/) help you save time and maintain consistency by reusing the structure of frequently repeated projects. Instead of recreating the same setup each time, you can save a project as a template and use it to quickly launch new projects with predefined configurations.

Templates are ideal for:

- **Standardized workflows** (e.g., onboarding, implementation, hiring)
- **Repeatable task structures** across clients or teams

Each project template can store:

- **Template Name**
- **Template Image**
- **Template Description**
- Optionally copy:
- Project Details
- Sections
- Tasks Details
- Subtasks Details

**How to Create a Project Template**

1. **Open the Project**

Navigate to the project you want to save as a reusable template.

1. **Click the Three-Dot Menu (⋮)**

Located next to the project name. From the dropdown, select **Save as Template**.

1. **Fill in Template Details**
1. **Template Name** (required): Give your template a clear name.
1. **Template Image** (optional): Upload an identifying image or icon.
1. **Description** (optional): Add a short summary about the purpose or usage.
1. **Template Type**: Defaults to **Project**.
1. **Choose What to Include in the Template**

**✅ Project Details to Copy**

Displayed as checkboxes. Select the elements to carry over into future projects:

- Client
- Priority
- Sprint Management
- Custom Fields
- People
- Dates

**🔀 Copy Sections (Toggle)**

Turn this ON to include your project’s section structure (e.g., "To Do", "Backlog", etc.).

Turn OFF to exclude them.

**🔄 Copy Tasks (Toggle)**

- When enabled, all project tasks will be included in the template.
- You’ll also see a set of **task-level fields** that will be copied along with each task:
- Status
- Priority
- Estimation
- Assignee
- Tags
- Custom Fields
- Dates

**📑 Copy Subtasks (Toggle)**

- This becomes available only if **Copy Tasks** is ON.
- If enabled, **subtasks** of copied tasks will also be included.
- 🔔 **Important:** The fields selected under **Copy Tasks** (like status, tags, etc.) will also apply to **subtasks**. Users do not currently have the option to select different fields for subtasks.
 **Click Save**

Once everything is configured, click **Save**.

Your custom project template is now available for reuse when creating new projects.
