## Overview 

The **Usage** tab under Workflows provides an overview of the total workflows configured, the number of workflow runs executed, the applicable monthly execution limits, and the breakdown of successful and unsuccessful runs. This helps you monitor workflow automation consumption and stay aligned with your plan limits.
## **Steps to Access Workflow Usage**
1. Log in to [**5day.io**](http://5day.io/).
1. Navigate to the **Project** where you want to check workflow usage.
1. Click on the **three-dot (⋮) icon** in the project header.
1. Click on **Settings**.
1. Inside Project Settings, click on **Workflows**.
1. Click on the **Usage** tab.

You will now see workflow configuration and execution details.

image

## **What You Can See in Workflow Usage**
The Usage section displays two key information cards:

### **1️⃣ Configured Workflows**
This section shows:

- Total workflows configured in the project
- The maximum workflows allowed as per your plan
- Current usage count (e.g., 20/50)
- Reset or refresh date (if applicable)

This represents the number of workflows you have created.
### **2️⃣ Workflow Runs (Executions)**
This section shows:

- Total workflow runs allowed under your plan
- Total workflow runs executed in the current cycle
- Number of **Successful Runs**
- Number of **Unsuccessful Runs**
- Reset date for the usage cycle

Every workflow execution is counted as a run, regardless of whether it is:

- Success
- Failed

Both successful and unsuccessful runs are included in the total run count.

Usage resets on a monthly basis as per your billing cycle.
