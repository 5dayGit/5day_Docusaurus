## Using [**5day.io**](http://5day.io/) as an Action in Zapier
## **Overview**
When you use [5day.io](http://5day.io/) as an **Action**, Zapier creates a new task in [5day.io](http://5day.io/) based on an event that occurs in another application.

Currently, the supported action event is:

- *Create Task*

This allows external activity to automatically generate work inside [5day.io](http://5day.io/).
## **When to Use This**
Use [5day.io](http://5day.io/) as an action when you want an external event to create a task in a specific project.

For example:

- Create a task when a new record is added in another system.
- Create a task when a form is submitted.
## **How It Works**
Inside Zapier:

1. Select another application as the Trigger.
1. Add a new Action step.
1. Choose [**5day.io**](http://5day.io/) as the Action app.
1. Select **Create Task**.
1. Connect your [5day.io](http://5day.io/) account.
1. Configure the required fields: 
   1. **Project** (required)
   1. **Task Name** (required)
1. Map additional optional fields if needed: 
   1. Task Type
   1. Task Status
   1. Assignee
   1. Start Date
   1. Due Date
   1. Section
   1. Priority
   1. Budget Type (only if Project Finance is enabled)
   1. Description
1. Test the action.
1. Publish the Zap.
