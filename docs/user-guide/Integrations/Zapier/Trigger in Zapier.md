## Overview
When you use [5day.io](http://5day.io/) as a **Trigger**, Zapier listens for an event inside [5day.io](http://5day.io/) and then performs an action in another application.

Currently, the supported trigger event is:

- **New Task**

This means whenever a new task is created in [5day.io](http://5day.io/), Zapier can automatically perform the next step in your workflow.
## **When to Use This**
Use [5day.io](http://5day.io/) as a trigger when you want activity inside [5day.io](http://5day.io/) to initiate automation in another system.

For example:

- When a new task is created, send a notification in another tool.
- When a task is created in a specific project, create a record in another application.
- When a task is assigned, trigger follow-up automation.
## **How It Works**
Inside Zapier:

1. Select [**5day.io**](http://5day.io/) as the Trigger app.
1. Choose the event **New Task**.
1. Connect your [5day.io](http://5day.io/) account.
1. Configure the required trigger field: 
   1. **Project** (required)
1. Optionally configure additional filters: 
   1. Section
   1. Task Type
   1. Task Status
   1. Priority
   1. Include Subtask (Yes / No)
   1. Assignee
1. Test the trigger.
1. Add the next Action step.
1. Publish the Zap.
## **Supported Trigger: New Task**
### **Event: New Task**
Triggers when a new task is created in [5day.io](http://5day.io/).
### **Supported Configuration Fields**
- Project (Required)
- Section
- Task Type
- Task Status
- Priority
- Include Subtask (Yes / No)
- Assignee
