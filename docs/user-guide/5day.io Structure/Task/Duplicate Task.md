# Duplicate Task

Duplicate Task creates an exact copy of any Task (with Subtasks, files, and settings) in one click - perfect for reusing templates, repeating sprints, or cloning client deliverables.

## Permission 
 
You need **Edit task permission** in the source Project and **Add task permission** in the target Project/Section.

## Ways to Access Duplicate Task

### List View / Board View

-   Select a Task → click **three dots** → **Duplicate**
-   To duplicate multiple tasks -> Select tasks -> **Bulk Action Bar** -> duplicate, all selected Tasks gets duplicated together in one click.

### Task Details

-   Open Task -> **Task Details** panel (right side) -> click **three dots** (top-right) -> **Duplicate**

### My Work

 -   Go to **My Work** -> find your Task -> click **three dots** -> **Duplicate**
 -   Works across all your Projects of workspace.

**Quick Note**
 - Task can be duplicated within Projects of same workspace.
 - Task cannot be duplicated to other workspace.
 - Subtask can be duplicated to same task only.

## Duplicate Task – What You Control

On clicking Duplicate, a smart panel opens:
| Field | Description |
|--|--|
| Task Name | Auto-filled with “Copy of [Original Name]” — editable before saving |
| Location | Choose: Same Project & Section (default) OR Any other Section OR Any other Project in the Workspace |
| Copy Options | Check on/off exactly what to bring over: Subtasks, assignee, priority, tags, discussions, dates, attachments, links |

<img width="3360" height="2300" alt="image" src="https://github.com/user-attachments/assets/8106b146-803e-4aa8-b907-fdaa413dbd9f" />

-   **Instant**: New duplicated task appears immediately — no refresh.
-   **Full fidelity**: All copied data stays linked (e.g., file attachments, discussion threads).
-   **History not copied**: Activity Stream starts fresh on the duplicate task.
-   **Subtasks included**: If checked, they duplicate with same nesting and settings.
- **Duplicate Subtask**: The subtask can be duplicated in the same task.
