## Overview

The Status field in [5day.io](http://5day.io/) helps track the progress of a task or subtask within a defined workflow.

Statuses provide visibility into where work stands, improve tracking across teams, and ensure tasks move through a structured lifecycle.

Both tasks and subtasks use statuses defined in the **Workspace Status Workflow**. The available statuses depend on the workflow mapped to the respective Task Type.
## **Status Workflow**
Task statuses are configured at the **Workspace level** under Status Workflow.

Statuses are grouped into workflow stages such as:

- New
- In Progress
- Done

Each stage can contain multiple statuses based on business needs.

By default, [5day.io](http://5day.io/) provides a **Default Workflow** with predefined statuses. Additional workflows can be created and customized based on team requirements.
## **Change Task or Subtask Status**
The status of a task or subtask can be updated from multiple locations across [5day.io](http://5day.io/).
## **Ways to Change Task Status**
### **Task / Subtask Details**
1. Open the Task or Subtask.
1. Locate the **Status field**.
1. Click on the current status.
1. Select a new status from the dropdown.
1. The status will be updated immediately.

image
### **List View / Board View**
### **List View**
1. Ensure the **Status column** is visible. 
   1. If not visible, click **Customize**.
   1. Enable the **Status field**.
   1. Save the view.
1. Click on the Status field against the task.
1. Select the required status.

image
### **Board View**
In Board View:

- Each column can also be represented as status.
- Drag and drop the task card from one column to another.
- The status updates automatically.

image
### **My Work**
1. Navigate to **My Work**.
1. Click on a task to open the Task Details.
1. Click on the **Status field**.
1. Select the desired status.

image
### **Bulk Update**
1. Select multiple tasks or subtasks.
1. Click on the **Bulk Update bar**.
1. Choose the **Status field**.
1. Select the required status.
1. Apply changes to update all selected items at once.

image
### **Quick Add (While Creating a Task)**
While creating a task using Quick Add:

1. Enter the task name and location.
1. Select the **Status** from the available list of the selected Task Type.
1. Create the task.

The task will be created with the selected status.

image
## **Status Behavior**
- The available statuses depend on the workflow mapped to the task type.
- Tasks and subtasks share the same workflow logic.
- Status configuration is managed at the Workspace level.
- Status changes reflect immediately across all views.
## **Quick Note**
- Statuses are managed through **Workspace → Status Workflow**.
- Changing a task’s status updates its progress across all project views.
- Status availability depends on the mapped workflow.
- Workflow modification requires Workspace Edit Permission.
