# Move Task
Move Task lets you instantly relocate any Task to a different Section or even a different Project - keeping history, files, and automations fully intact.

## Permission  
You should have **Edit Permission** in Task under **Roles and Permission.**
Below are further details on Edit permission.

|Permission Level  | Description |
|--|--|
|All |Applies to **all tasks** in the project, regardless of ownership. |
|Own  |Applies only to tasks where user is **Task Owner**. |
|Related  |Applies to tasks in which user is added as **Project Owner** or **Project member**  |
|None  |The user does **not have any access** for the selected action  |

## Ways to Access Move
### List View / Board View

-   Select a Task → click **three dots** → **Move**
-   To move multiple tasks -> Select tasks -> **Bulk Action Bar** -> Move, All selected Tasks move together in one click.

### Task Details

-   Open Task -> **Task Details** panel (right side) -> click **three dots** (top-right) -> **Move**

### My Work

 -   Go to **My Work** -> find your Task -> click **three dots** -> **Move**
 -   Works across all your Projects of workspace.

**Quick Note**

 - Task can be moved within Projects of same workspace.
 - Task cannot be moved to other workspace.
