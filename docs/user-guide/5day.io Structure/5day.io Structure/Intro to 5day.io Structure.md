## Overview
[5day.io](http://5day.io/) is organized using a simple structure that helps teams plan, organize, and execute work clearly. Each level in the structure builds on the one above it, so your work stays grouped and easy to track as your team scales.
### [**5day.io**](http://5day.io/) **Structure**
In [5day.io](http://5day.io/), work is organized like this:

**Workspace → Space → Project → Section → Task → Subtask**
### **What Each Level Represents**
- **Workspace:** The highest level used to manage work for an organization or major business unit.
- **Space:** A grouping inside a workspace, typically used for departments or focus areas.
- **Project:** A place where related tasks are managed under a single goal or initiative.
- **Section:** A way to group tasks inside a project (phases, sprints, milestones, etc.).
- **Task:** The main unit of work tracked in a project.
- **Subtask:** A smaller unit of work under a task to break execution into steps.
### **Why Structure Matters**
Using the [5day.io](http://5day.io/) structure helps you:

- Organize work by teams and initiatives
- Keep related tasks and projects grouped logically
- Track execution without losing context
- Scale from a single team to multiple departments easily
### **Quick Note**
- Every **task** belongs to a **project**.
- Every **subtask** belongs to a **task**.
- Workspaces are independent, and data is not shared across workspaces.
